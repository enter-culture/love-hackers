# PRD: 로테이션 소개팅 연습 앱

## 개요

소개팅 자리에서 말을 잘 못하거나 어색한 사람들을 위한 **로테이션 소개팅 연습 시뮬레이터**.
나는솔로 스타일 인트로 → 동물의 숲 분위기 3D 씬 → 실제 소개팅처럼 여러 사람과 대화 연습 → 호감도 결과 확인.

---

## 유저 플로우

```
인트로 화면 (/)
  → (구글 로그인 버튼 클릭)
로비 화면 (/lobby)
  → (인원수 선택 → 시작하기)
게임 화면 (/game) ← 3D 자유이동 월드
  ├─ 플레이어가 WASD로 이동
  ├─ NPC에게 접근 → "대화하기" 버튼 표시
  ├─ 대화 시작 → 하단 다이얼로그 UI (질문 + 객관식)
  ├─ 대화 완료 → NPC에 완료 마크 표시
  └─ 모든 NPC 대화 완료 → /result 이동
결과 화면 (/result)
  → 캐릭터별 호감도 점수
```

---

## 화면 스펙

### 1. 인트로 화면 `/`

**레이아웃**
- 전체 화면 다크 배경
- 상단 절반: Lottie 애니메이션 (나는솔로 스타일 — 하트, 반짝임, 두근거림 느낌)
- 하단: 앱 타이틀 + 구글 로그인 버튼

**동작**
- Lottie는 자동 재생, 루프
- 구글 로그인 버튼: 실제 인증 없음, 클릭 시 `/lobby` 로 이동
- 버튼 텍스트: "Google로 시작하기"

**기술**
- `lottie-react` 사용
- Lottie 파일: LottieFiles에서 "love", "heart", "dating" 키워드 무료 JSON 다운로드

---

### 2. 로비 화면 `/lobby`

**레이아웃**
- R3F(React Three Fiber)로 동물의 숲 스타일 3D 씬 전체 화면
- 씬 구성:
  - 낮 시간대 하늘 (연한 하늘색 그라디언트)
  - 카페 or 공원 느낌 바닥 (초록색 잔디 or 나무 바닥)
  - 귀여운 동물 캐릭터들이 테이블에 앉아 대기 중 (인원수에 따라 표시)
  - 카메라는 약간 위에서 내려다보는 아이소메트릭 뷰
- UI 오버레이 (HTML):
  - 상단: "오늘의 소개팅 파티" 타이틀
  - 중앙 하단: 인원수 선택 버튼 (2명 / 3명 / 4명 / 5명)
  - 선택 버튼 아래: "시작하기" 버튼

**동작**
- 인원수 선택 시 해당 숫자만큼 캐릭터가 씬에 나타남
- "시작하기" 클릭 → `/game` 로 이동, 인원수를 URL searchParam으로 전달 (`?count=3`)

**기술**
- `@react-three/fiber`, `@react-three/drei`
- Three.js BoxGeometry + MeshToonMaterial 로 셀 쉐이딩 스타일 캐릭터
- `OrbitControls` disabled, 고정 카메라
- 배경은 CSS gradient로 처리 (Canvas 밖)

---

### 3. 게임 화면 `/game`

#### 3-1. 3D 월드

**씬 구성**
- 카페 or 공원 배경 — 동물의 숲 스타일
- 플레이어 캐릭터 (박스형 귀여운 캐릭터) — 유저가 직접 이동
- NPC 캐릭터들 — 각자 지정 위치(테이블)에 고정 배치
- 바닥: 격자형 타일 or 잔디 평면
- 하늘: 연한 하늘색 배경

**카메라**
- 3인칭 — 플레이어 뒤 약간 위에서 따라다니는 카메라
- 카메라는 항상 플레이어를 바라봄
- 줌/회전 없음 (고정 각도)

#### 3-2. 플레이어 이동 (클릭 투 무브)

**이동 방식**
- 바닥(Plane Mesh) 위 **마우스 클릭 → 해당 위치로 캐릭터 이동**
- Raycasting으로 클릭 지점의 3D 좌표를 계산
- 캐릭터는 클릭 지점까지 부드럽게 이동 (lerp)
- 이동 중 클릭 지점 방향으로 캐릭터 회전
- 목적지 도달 시 정지

**클릭 피드백**
- 클릭한 바닥 위치에 잠깐 원형 리플 이펙트 표시 (CSS or Three.js ring)

**제약**
- 대화 중에는 클릭 이동 비활성화
- 맵 경계 밖 좌표는 clamp 처리

#### 3-3. NPC 상호작용

**접근 감지**
- 플레이어와 NPC 간 거리 < 2.5 유닛이면 상호작용 가능 상태
- 상호작용 가능 상태: NPC 위에 `"💬 대화하기"` 말풍선 아이콘 표시 (HTML 오버레이, `useThree`의 camera로 3D→2D 좌표 변환)
- 이미 완료된 NPC: `"✅"` 아이콘 표시, 재대화 불가

**대화 시작**
- 접근 시 스페이스바 or 버튼 클릭 → 대화 모드 진입
- 대화 중 플레이어 이동 입력 비활성화

#### 3-4. 대화 UI (HTML 오버레이)

**레이아웃** (화면 하단 고정)
```
┌─────────────────────────────────────────┐
│ [NPC 이름]                              │
│ "주말엔 주로 뭐 해요?"                   │  ← 타이핑 효과
└─────────────────────────────────────────┘
  [ 집에서 쉬어요 ]  [ 친구랑 놀아요 ]  [ 여행 가요 ]
```

**동작 흐름**
1. 대화 진입 → NPC 이름 + 질문 타이핑 효과로 표시
2. 타이핑 완료 → 선택지 3개 버튼 등장 (fade in)
3. 선택지 클릭 → 짧은 NPC 반응 표시 ("오, 그렇구나!", "나도 그래요!" 등)
4. 반응 1.5초 후 → 다음 질문 (총 3문항)
5. 3문항 완료 → 대화 UI 닫힘, NPC에 ✅ 표시, 이동 재활성화
6. 모든 NPC 완료 → "모두와 이야기가 끝났어요! 결과 보기" 버튼 표시 → `/result`

**진행 표시 (상단 HTML 오버레이)**
- `완료 2 / 3명` 형태로 표시

**질문 데이터** (하드코딩)
```ts
const QUESTION_BANK = [
  { question: "주말엔 주로 뭐 해요?", choices: ["집에서 쉬어요", "친구랑 놀아요", "혼자 여행 가요"] },
  { question: "좋아하는 음식 종류는?", choices: ["한식", "양식", "아시안 퓨전"] },
  { question: "이상형의 첫인상은?", choices: ["눈이 마주쳤을 때 끌려요", "목소리가 중요해요", "분위기로 느껴요"] },
  { question: "여행 스타일은?", choices: ["계획형", "즉흥형", "반반"] },
  { question: "카페에서 주로 뭐 마셔요?", choices: ["아메리카노", "라떼류", "논커피"] },
  { question: "쉬는 날 하고 싶은 건?", choices: ["드라이브", "집에서 영화", "맛집 탐방"] },
]
// 각 NPC마다 QUESTION_BANK에서 3문항 순서대로 배정
```

**호감도 계산**
- 선택지마다 숨겨진 점수 (0 / 5 / 10) 사전 배정
- NPC별 합산 (최대 30점)

**기술**
- 단일 페이지 `/game`, 내부 상태로 대화 진행
- 상태: `useGameState` custom hook (플레이어 위치, 대화 상태, 점수)
- 키보드 입력: `useEffect` + `keydown/keyup` 이벤트
- 3D→2D 좌표 변환: `@react-three/drei`의 `Html` 컴포넌트 (3D 오브젝트에 HTML 붙이기)
- 타이핑 효과: `useTypewriter` custom hook

---

### 4. 결과 화면 `/result`

**레이아웃**
- 배경: 동물의 숲 씬 (모든 캐릭터 한 화면에)
- 상단: "오늘의 소개팅 결과" 타이틀
- 캐릭터 카드 목록 (가로 스크롤 or 세로 나열):
  - 캐릭터 이름
  - 3D 캐릭터 아이콘 (혹은 색깔 아바타)
  - 하트 게이지 바 (0~100%)
  - 점수 숫자 (예: 23 / 30)
  - 한줄 평 (점수 구간별)
    - 25~30: "완전 잘 통했어요 💕"
    - 15~24: "괜찮은 첫만남이었어요 😊"
    - 0~14: "좀 더 자연스럽게 해봐요 😅"
- 하단: "다시 해보기" 버튼 → `/lobby` 로 이동

| 항목 | 선택 |
